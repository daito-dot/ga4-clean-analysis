GA4 クリーンデータと実ページの構造を突き合わせた CRO ディープ分析を実行してください。

`skills/cro-deep-analysis/SKILL.md` のワークフローに **厳密に** 従い、以下を順番に実行します:

## 実行手順

### Phase 1: GA4 クロスディメンション分析
CLAUDE.md の確定済みフィルタを使い、8つのレポートを**並行実行**:
- デバイス×ランディングページ（直帰率ギャップ特定）
- チャネル×イベント（チャネル別CV行動差）
- デバイス×イベント（デバイス別CV行動差）
- 流入元別の質スコア
- ページ×イベント（CTA導線の死角）
- ページ別基本指標
- デバイス別基本指標
- チャネル別基本指標

### Phase 2: データクロス分析
6つの必須チェック項目を分析し、数値に基づく発見を抽出

### Phase 3: 実ページ取得
問題のあるページを WebFetch で取得し、構造・CTA・フォームを記録

### Phase 4: データ×UX 突き合わせ
Phase 2 の数値と Phase 3 のページ構造を突き合わせ、原因を特定

### Phase 5-6: 改善提案の構造化・出力
ICEスコア付きの具体的な改善提案を生成

## 重要

- **数値を整形し直すだけの分析は禁止**。必ずクロス分析で発見を抽出すること
- **抽象的な提案は禁止**。「CTAを改善」ではなく「ボタン文言を○○→○○に変更」のレベルで書くこと
- **実ページを見ずに提案を書くことは禁止**。必ず WebFetch で実際のページ構造を確認すること
- `skills/marketing-cro/SKILL.md` のベンチマーク・フレームワークを判断基準として参照すること

### Phase 7: レポート出力
分析完了後、自動的に Word レポート（.docx）を生成する。

1. `skills/cro-deep-analysis/REPORT_TEMPLATE.py` をベースに、Phase 2 の分析データを `DATA` dict に、Phase 4-6 の文章を `NARRATIVES` dict に注入する
2. venv で `python-docx`, `matplotlib`, `numpy` をインストールして実行
3. 出力先: `{プロジェクトルート}/{サイト名}_CRO分析レポート.docx`
4. グラフ仕様は `skills/cro-deep-analysis/CHART_SPEC.md` に準拠

ユーザーが引数でサイトURLやデータを指定した場合は、それを使用してください。
指定がない場合は CLAUDE.md の対象サイト情報を使用してください。
