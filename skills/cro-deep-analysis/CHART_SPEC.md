# CROディープ分析 — グラフ仕様書

このドキュメントは、CRO分析レポートのExcel出力で生成する**全グラフの仕様**を定義する。
各グラフは「作業者が何を判断できるか」を起点に設計されている。

## 設計原則

1. **判断ファースト**: 「きれいな図」ではなく「次に何をすべきかわかる図」
2. **損失の可視化**: 問題の大きさを「失われたリード数」「失われた売上」で表現
3. **ベンチマーク対比**: 業界水準と自サイトのギャップを常に示す
4. **アノテーション必須**: 数値の意味をグラフ内に直接書く。凡例だけでは判断できない

---

## グラフ一覧

### Graph 1: 損失リードマップ（最重要）

**目的**: サイト全体で「どこで何件のリードが失われているか」を1枚で把握する
**判断**: どの問題から着手すべきか、投資対効果の大きい箇所はどこか

**データ**:
- 各ボトルネック箇所（フォーム離脱、CTA死角ページ、モバイル断裂等）
- 各箇所の推定損失リード数/月
- 改善後の期待リード数/月

**形式**: 横棒グラフ（Waterfall風）
- X軸: 損失リード数/月
- Y軸: ボトルネック箇所（損失が大きい順）
- 色: 赤=現在の損失、緑=改善後の期待回復
- アノテーション: 各棒に「現在→改善後」の数値と改善施策名

**損失リード数の算出ロジック**:
```
損失 = 対象トラフィック × (ベンチマークCVR − 現在CVR)

例: CTA死角ページの損失
= 635PV × (3%ベンチマーク − 0%現在) = 月19件のリード損失

例: モバイルフォーム断裂
= demo_complete 15件 × (30%ベンチマーク完了率 − 0%現在) = 月4.5件
```

**ベンチマーク出典**: marketing-cro/SKILL.md のConversion Rate Benchmarks
- ランディングページ: 2-3% (Average)、4-5% (Good)
- フォーム完了率: 30-40% (Average)、45-55% (Good)

### Graph 2: ファネルウォーターフォール（デバイス別）

**目的**: ファネルの各ステップで「何人が残り、何人が落ちたか」を視覚的に追跡
**判断**: どのステップが最大のボトルネックか、Desktop/Mobileで断裂箇所が違うか

**データ**:
- GA4 Report #3（deviceCategory × eventName）からイベント数を抽出
- ファネル順: page_view → scroll → cta_click → demo_start → demo_complete → generate_lead → form_start → form_submit

**形式**: 2段構成（上: Desktop、下: Mobile）のウォーターフォール
- 各ステップの残存数を棒で表示
- ステップ間の離脱数を赤い矢印 + 数値で表示
- 離脱率が50%を超えるステップは背景を赤くハイライト
- 「致命的断裂」ポイントに⚠マーク + 説明テキスト

**アノテーション例**:
```
[demo_complete 15] ──(87%離脱! 13人消失)──▶ [generate_lead 2]
                     ↑
              フォーム摩擦が原因。7フィールド→3に削減で回復見込み
```

### Graph 3: CTA導線ヒートマップ

**目的**: どのページにどのCV導線があり/ないかを一覧で判断
**判断**: CTAを追加すべきページの優先順位

**データ**:
- GA4 Report #5（pagePath × eventName）からCVイベントの有無
- 対象イベント: cta_click, generate_lead, form_start, form_submit, demo_start, demo_complete

**形式**: マトリクスヒートマップ
- 行: ページ（PV降順）
- 列: CVイベント種別
- セルの色: 0件=赤、1-5件=黄、6件以上=緑
- 右端列: そのページの合計PV（棒グラフ併記）

**判断基準**:
- 赤い行（全イベント0）でPVが高い = 最優先でCTA追加
- 黄色い行 = CTAはあるが効率が悪い、改善余地あり

---

### Graph 4: フォーム離脱ファネル

**目的**: /contactページ内でのユーザー行動を段階的に可視化
**判断**: フォームのどの段階で最大の離脱が起きているか

**データ**:
- GA4 Report #5の/contactデータ: page_view → scroll → generate_lead → form_start → form_submit
- 各ステップのユーザー数

**形式**: ファネル図（上が広く下が狭い台形）
- 各段の幅がユーザー数に比例
- 段と段の間に「離脱率 XX%（YY人離脱）」を赤字で表示
- 最大離脱ポイントを強調（太枠 + 背景色）
- 右側に「改善施策」をテキストで併記

**アノテーション**:
```
ページ閲覧 36人 ━━━━━━━━━━━━━━━━━━
                    ↓ 44%離脱 (16人)
generate_lead 20人  ━━━━━━━━━━━━━
                    ↓ ★90%離脱 (18人)  ← ここが最大ボトルネック
form_start 2人      ━━━
                    ↓ 0%
form_submit 2人     ━━━

★の改善策: 7フィールド→3フィールド、マルチステップ化
```

### Graph 5: チャネル品質マトリクス

**目的**: どの流入元が「質の高いトラフィック」を送っているか判断
**判断**: どのチャネルに投資を増やし、どのチャネルのLP/導線を改善すべきか

**データ**:
- GA4 Report #4（sessionSourceMedium）: sessions, bounceRate, avgDuration, keyEvents
- CVR = keyEvents / sessions

**形式**: バブルチャート（散布図）
- X軸: 平均滞在時間（秒）— エンゲージメントの深さ
- Y軸: CVR (%) — コンバージョン効率
- バブルサイズ: セッション数 — トラフィック量
- バブル色: 4象限で色分け
  - 右上（高滞在×高CVR）= 緑 → 拡大すべき
  - 左上（低滞在×高CVR）= 青 → ウォームトラフィック、効率的
  - 右下（高滞在×低CVR）= 黄 → エンゲージメントあるがCV導線不足
  - 左下（低滞在×低CVR）= 赤 → 改善or撤退

**4象限ラベル**:
```
        高CVR
    ┌─────────┬─────────┐
    │ 効率的   │ 理想的   │
    │ (拡大)   │ (最大化) │
    ├─────────┼─────────┤
    │ 要改善   │ CTA不足  │
    │ (撤退?)  │ (導線修正)│
    └─────────┴─────────┘
   短い滞在          長い滞在
```

### Graph 6: デバイス×ページ直帰率ギャップ

**目的**: Desktop/Mobileで直帰率に大きな差があるページを特定
**判断**: モバイル最適化が必要なページの優先順位

**データ**:
- GA4 Report #1（deviceCategory × landingPagePlusQueryString）の直帰率

**形式**: ペアバーチャート（横棒、ページごとにDesktop/Mobile並列）
- 15%以上のギャップがあるペアを赤枠でハイライト
- ギャップの数値をアノテーション
- 直帰率50%以上の棒には「危険」ラベル
- 直帰率100%の棒には「即離脱」ラベル

**判断基準**（marketing-cro参照）:
- ランディングページ直帰率 40%以上 = 要改善
- Desktop/Mobileギャップ 15%以上 = モバイル最適化必須

---

### Graph 7: ICE優先度マトリクス

**目的**: 全施策の優先順位を1枚で判断
**判断**: 今週何から着手するか

**形式**: 横棒グラフ（積み上げ）
- Y軸: 施策名（ICEスコア降順）
- X軸: スコア (0-10)
- 3色積み上げ: Impact（赤系）、Confidence（青系）、Ease（緑系）
- 右端にICE合計スコアをラベル表示
- ICE 8以上 = 緑背景ラベル「今すぐ」
- ICE 7-8 = 黄背景「今月中」
- ICE 7未満 = 灰背景「来月以降」

**補足テキスト**: 各施策の1行サマリー（「7フィールド→3フィールドに削減」等）
